# STEP 03 — Internationalisation (react-i18next, Uzbek-first)

## Prerequisite
Master prompt loaded. Steps 01–02 complete.

## Goal
Wire react-i18next so every user-facing string flows through `t('key')`. Uzbek (`uz`) is the default and fallback. Russian (`ru`) and English (`en`) files exist but are mostly empty — UZ fallback fires for missing keys. Seed the `common` namespace with the keys that every feature will need.

## Deliverables
- `dashboard/src/i18n/index.ts` — i18next config
- `dashboard/src/i18n/locales/uz.json` — populated with `common.*` + initial app shell keys
- `dashboard/src/i18n/locales/ru.json` — empty objects (`{}` per namespace)
- `dashboard/src/i18n/locales/en.json` — empty objects (`{}` per namespace)
- `dashboard/src/i18n/uz-locale.ts` — date-fns Uzbek locale wrapper + `formatDate`, `formatNumber`
- Import in `main.tsx` so i18n is initialised before React renders
- A working demo on the home placeholder showing localised strings

### 2. Create `src/i18n/index.ts`

```ts
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';
import LanguageDetector from 'i18next-browser-languagedetector';

import uz from './locales/uz.json';
import ru from './locales/ru.json';
import en from './locales/en.json';

void i18n
  .use(LanguageDetector)
  .use(initReactI18next)
  .init({
    resources: {
      uz: uz,
      ru: ru,
      en: en,
    },
    fallbackLng: 'uz',
    supportedLngs: ['uz', 'ru', 'en'],
    defaultNS: 'common',
    ns: ['common', 'dashboard'],
    interpolation: {
      escapeValue: false, // React already escapes
    },
    detection: {
      order: ['localStorage', 'navigator'],
      lookupLocalStorage: 'devon.dashboard.lang',
      caches: ['localStorage'],
    },
    returnNull: false,
  });

export default i18n;
```

### 5. Date and number helpers — `src/i18n/uz-locale.ts`

```ts
import { format as fnsFormat, formatDistanceToNow as fnsDistance, Locale } from 'date-fns';
import { uz, ru, enUS } from 'date-fns/locale';
import i18n from './index';

const localeMap: Record<string, Locale> = { uz, ru, en: enUS };

export function formatDate(date: Date | string, pattern = 'dd.MM.yyyy'): string {
  const d = typeof date === 'string' ? new Date(date) : date;
  const lang = i18n.language;
  const effectivePattern = lang === 'en' ? 'MMM d, yyyy' : pattern;
  return fnsFormat(d, effectivePattern, { locale: localeMap[lang] ?? uz });
}

export function formatDateTime(date: Date | string): string {
  const d = typeof date === 'string' ? new Date(date) : date;
  const lang = i18n.language;
  const pattern = lang === 'en' ? "MMM d, yyyy 'at' h:mm a" : 'dd.MM.yyyy HH:mm';
  return fnsFormat(d, pattern, { locale: localeMap[lang] ?? uz });
}

export function formatRelative(date: Date | string): string {
  const d = typeof date === 'string' ? new Date(date) : date;
  return fnsDistance(d, { addSuffix: true, locale: localeMap[i18n.language] ?? uz });
}

export function formatNumber(n: number, options?: Intl.NumberFormatOptions): string {
  const lang = i18n.language;
  const localeCode = lang === 'en' ? 'en-US' : `${lang}-UZ`;
  return new Intl.NumberFormat(localeCode, options).format(n);
}
```

## Notes

- **Key naming discipline.** Every new string MUST be a key, never a literal in JSX. Subsequent step prompts assume this.
- **Adding keys.** Each feature step adds keys to `uz.json` under `dashboard.<feature>.*`. Keep the file sorted alphabetically within each namespace.
- **Pluralisation.** Use ICU plural format. Russian needs `one/few/many/other`; English needs `one/other`; Uzbek typically uses `one/other`. Test plurals in `ru.json` once translations exist.
- **Language switcher.** Out of scope for the demo. The product roadmap (v1.1) adds Russian — at that point the user menu gets a language toggle.

## What "done" looks like

Every string in `App.tsx` is sourced from `t('key')`. Adding new translations is purely a JSON edit. The infrastructure is ready for every subsequent feature step.
